## ⭐ Best demo example for the room

**Use this:** Live coding-agent run on the cheapest model, with Bongo wrapping the loop. On stage: give a cheap-model agent (e.g. GPT-4o-mini / Haiku / a small open model via HF Inference Providers ":cheapest") a real multi-step coding task — "fix this failing test suite." Run it TWICE side by side. Left (no Bongo): the cheap agent at step 2 emits a subtly broken edit (wrong file path, malformed tool-call arg, or code that imports an undefined variable) — schema-valid, no error thrown — and the bug silently poisons the next 6 steps; the run ends red (tests still fail) after burning N tokens. Right (Bongo in the loop): Bongo traces each step and runs ONE cheap verifier signal per step that is deterministic and instant — does the code execute / does the test runner go green / is the tool-call JSON well-formed? The moment step 2 fails verification, Bongo retries-or-escalates THAT ONE step to a strong model, the run goes green. Then flash the bill: right side paid cheap-model price on ~90% of steps + one escalation, left side paid less but shipped a broken result. Close on the live cost-comparison panel: "on this step, provider X at 1/10th the cost was indistinguishable; on THAT step it wasn't — here's where you overpay." The verification is the whole pitch: a 50-step trajectory judged in one ~$0.001 call, and for code the pass/fail is free and deterministic — no hand-waving on stage.


**Why it resonates:** Three reasons it wins this exact room. (1) Universality: every team at a YC×agents hackathon has built or debugged a coding/tool-using agent, and the compounding-error math (95% per-step = ~60% over 10 steps) is the slide the whole room already nods at — no problem to explain. (2) Cheap verifiability on stage: code is the one domain where the verifier is deterministic and nearly free — the test runner is ground truth, so the "cheaply verify each step" claim is demonstrated, not asserted; you avoid the credibility trap of "how do you verify without the expensive model?" (3) Judge fit: it maps 1:1 onto a shipping product at FOUR of the judge orgs (Datadog Bits Dev Agent watches CI and iterates until green; Mistral Vibe Remote Agents open PRs overnight and already have GitHub-filed malformed-tool-call/context-loss bugs; HF smolagents/Open Deep Research whose own debugging advice is the blunt "use a more powerful LLM"; and Genesis's reward-CODE generation where ~4% runs-but-wrong is their documented pain). The per-step escalation directly replaces each company's current blunt instrument (escalate the whole run / "just use a bigger model"), and the cross-provider cost panel hits Datadog's "69% run 3+ models, no neutral arbiter" and HF's router (cost exposed, reliability-per-dollar not).


**Universal framing line:** "Every agent in this room runs on the cheapest model that 'mostly works' — and at 95% per-step accuracy a 10-step agent is only 60% reliable end-to-end, failing silently: schema-valid, no error, just wrong. Bongo traces each step, cheaply verifies it, and escalates ONLY the failing step to a stronger model — across any provider. You keep the cheap model's price and get the expensive model's reliability, without rewriting your agent."


**Backup example:** RAG/support-bot grounding check: cheap model drafts an answer from thin retrieved context and ships a confident hallucination (cheaper models produce "plausible answers even with inadequate context"); Bongo inserts a cheap "is this grounded in the retrieved docs?" verify and escalates only ungrounded answers to the strong model. Universal and resonant (Datadog AppFolio Realm-X inbox at hundreds of thousands of msgs/day; QRT filing extraction; Mistral KYC/support triage), but slightly weaker as a LIVE demo because grounding verification is fuzzier on stage than a green/red test runner — keep it as the backup if the room skews non-coding.


**Reality-check on these examples:**

- realistic: **partly** — concerns: DEMO STAGING / NON-DETERMINISM IS THE BIGGEST RISK. The pitch's whole punch is 'cheap model emits a subtly broken edit at step 2, live.' But cheap LLMs are non-deterministic even at temperature=0 (tie-breaking, batch/hardware nondeterminism on hosted inference, GPT-4o-mini/Haiku via APIs) per arxiv 2408.04667 and mikulskibartosz.name/why-temperature-0-isnt-deterministic. You cannot reliably guarantee the failure fires at step 2 in front of YC + Datadog/Mistral/QRT engineers. If the left run accidentally succeeds, the pitch collapses. Most 'live' demos like this are effectively scripted/recorded; sophisticated judges sniff that out.; THE VERIFIER STORY IS STRONGER FOR CODE THAN THE PITCH ADMITS, AND THAT'S BOTH GOOD AND A TRAP. The defensible signals are DETERMINISTIC and ~free: JSON/schema validation, tool-call format checks, does-it-execute, test-runner green/red (confirmed: blog.jztan.com/ai-agent-error-handling-patterns, futureagi tool-chaining, and 'deterministic checks catch the most common hallucinations at essentially zero cost'). But the pitch ALSO claims a generic '50-step trajectory judged in one ~$0.001 LLM call.' That cheap LLM-judge claim is the weak link: Hard2Verify (arxiv 2510.13744) shows weak/cheap verifiers mark nearly every step correct, and ~93% of teams struggle to make LLM judges reliable (calibration/drift/bias). Judges from Datadog (which ships LLM-as-judge evals) will know cheap judges are unreliable. Lead with the deterministic code signal; do NOT oversell the universal cheap-judge.; DIFFERENTIATION VS EXISTING ROUTERS. Per-request/per-conversation routing is commoditized: OpenRouter Auto (NotDiamond-powered), Vercel AI Gateway (GA Aug 2025), Martian. Cascades with self-generated-test thresholds and per-step escalation are also published (C3PO arxiv 2511.07396, Aragog ~98% cost cut, ABC, the cascade survey arxiv 2603.04445). Nicolas Dessaigne (ex-Algolia founder, infra-savvy) will immediately ask 'how is this not OpenRouter + a verifier loop?' Your real wedge is narrow and must be stated explicitly: existing routers decide PRE-execution by predicted difficulty and PIN the model for the whole conversation; Bongo escalates a SINGLE step POST-hoc, triggered by an observed verification failure, mid-trajectory. That distinction is genuine but easy to miss.; AUDIENCE-FIT FRICTION. The 'fix this failing test suite' task is realistic and credible (SOTA agents still fail >20% of SWE-bench Verified as of early 2026), so it's not a toy. Datadog already does agent tracing + online evals + escalation insights; Mistral ships an Agents API with a code-interpreter sandbox. To them, 'trace each step + cheap verifier + escalate' reads as features they partially have. (1) DE-RISK THE STAGE: do not rely on live non-determinism. Either pre-record the two runs (and say so), or pin a fixed seed AND hard-rig the failure — give the cheap agent a frozen trajectory/cached step-2 output that is a known schema-valid-but-broken edit (undefined-variable import or wrong path). Have a deterministic fallback recording. The failure must be guaranteed, not hoped for. (2) MAKE THE VERIFIER DETERMINISTIC, NOT AN LLM JUDGE: for the code demo the per-step signal is 'does it execute / does pytest go green / is the tool-call JSON well-formed' — free, instant, unimpeachable. (3) NAIL THE ONE-LINE WEDGE vs routers, say it on stage before a judge asks: 'OpenRouter/Vercel route per-request by predicted difficulty and lock the model for the whole run; Bongo escalates ONE step, AFTER a verification signal fails, mid-trajectory, across providers a single lab can't reach.' ## Per-company deep dive


### Datadog  _(confidence: high)_

**🟢 Bongo example:** "A Datadog engineer runs Bits AI SRE on a Kafka latency alert. The agent fires 12 tool calls; one finds the real cause (commit-latency spike) but a noisy 'upstream critical errors' signal drags the final summarization step to the WRONG root cause — the exact failure they had to re-architect around. Bongo sits on that summarization step: it cheaply verifies the conclusion against the supporting telemetry, catches that the named root cause isn't causally backed by the evidence, and on that one step escalates from the cheap model to a stronger one (or asks for the missing causal trace) — instead of paying frontier-model prices for all 12 calls. You ship the cheap-model investigation at frontier-model trustworthiness, and only pay up where the step actually needed it."


**Agentic workflows they run:**
- Hypothesis-driven incident RCA: Bits AI SRE recursively generates root-cause hypotheses, validates each with a targeted tool call, and prunes branches. A documented failure: an earlier Kafka investigation fired 12 tool calls; one correctly found the commit-latency root cause but other calls surfaced noisy 'critical errors in an upstream service,' and the summarization step returned the WRONG root cause. They re-architected to filter for causal relevance — a per-step verification problem at its core.
- Self-evaluating agent loop: hundreds of Datadog teams contributed labeled real incidents; an LLM judge scores Bits' conclusions against ground-truth root-cause labels to track and improve accuracy over time.
- Autonomous code-fix loop: Dev Agent generates a fix -> opens PR -> watches CI -> iterates until green. Each iteration is more LLM calls; a weak first-pass fix means more retry loops (more cost, more latency).
- Batch vulnerability remediation: define a class of vulns (e.g. all SQLi) across many repos and the agent fixes them all and groups them into PRs — LLM reasoning fanned out across thousands of code sites.

**Cost / reliability pain:**
- Datadog's own State of AI Engineering 2026 data: in Feb 2026, 5% of all LLM call spans errored and 60% of those were rate-limit errors; in Mar 2026, ~8.4M rate-limit errors — they call rate limits 'the single most common source of agent failure.' Provider capacity ceilings directly force reliability compromises.
- Token cost is exploding: median customer's tokens-per-request more than doubled YoY, 90th-percentile power users quadrupled; system prompts are 69% of input tokens and only 28% of calls use prompt caching — i.e. most calls re-process the full prompt and overpay.
- Context-window blowup in agents: early Bits SRE scaled input tokens linearly with tool calls, exhausting context and degrading accuracy — a direct cheap-vs-expensive-vs-correct tradeoff in agent design.
- Silent wrong answers, not crashes: the Kafka miscS where the agent confidently returned the wrong root cause is the dangerous failure mode — it passes, it's just wrong. On-call trust is the whole product.

For a systematic fund, a backtest that compiles but is financially meaningless is the worst failure mode — it looks fine until it loses money live.
- Extraction errors at scale that poison signals: even strong stacks land ~94% text / ~91% table accuracy (intuitionlabs.ai); the wrong 6-9% of extracted numbers silently corrupts a factor computed across thousands of filings, and the error is invisible until alpha decays.
- Cheap-vs-expensive token economics on huge volume: 2026 prices span ~$0.10/M (nano/cheap) to ~$30/M (frontier reasoning) input tokens (cloudzero, costgoat). Running frontier models on every one of millions of filings/transcripts is the obvious-but-expensive path; running a cheap model risks reliability. This is exactly Bongo's sweet spot.
- Audit/reproducibility burden: 'Every AI-assisted investment decision needs an audit trail that would survive an SEC exam' — versioned prompts, model outputs, and rationale (tommasomariaricci.com). Ad-hoc LLM calls with no per-step trace are a compliance liability.

### Generic YC / early-stage startups building AI products on LLM APIs (the hackathon room itself)  _(confidence: high)_

**🟢 Bongo example:** "Point at any team in the room running an agent: 'Your coding agent does a 50-step run on the cheapest model to save money. Step 2 passes a wrong file path, schema-valid, no error thrown — and it silently poisons all 48 steps after it. You find out from a failed demo, or a $4,200 weekend bill where you can't even see which step burned it. Bongo traces each step, runs a 1-call cheap verifier over the whole trajectory for a fraction of a cent, catches step 2 the moment it goes wrong, retries it — and ONLY that step — on the strong model. You keep the cheap model's price and get the expensive model's reliability, across any provider. That's a 95%-reliable 10-step workflow instead of a 60%-reliable one, without rewriting your agent.'"


**Agentic workflows they run:**
- Support/RAG answer pipeline: embed query -> vector search -> stuff context -> cheap model drafts answer -> (today: ship it) -> customer gets a confident wrong answer when retrieval was thin. Bongo inserts a cheap verify (is this grounded in retrieved docs?) and escalates only ungrounded answers to the strong model.
- Document/data extraction pipeline: PDF/email/invoice -> cheap model -> JSON -> Pydantic/Zod schema validation -> downstream DB write. The schema check passes but the field value is wrong; a wrong value at step 2 silently corrupts every downstream step. This is 'the most common and most insidious production failure mode.'
- Autonomous coding agent: plan -> read files -> edit -> run tests -> repeat for 50-200 steps, full context re-sent each loop. One bad tool argument early cascades through the whole run; teams discover it only at the end (or via a $4,200 weekend bill).
- Multi-agent / orchestrated workflow: agent A's output feeds agent B feeds C; 'each subsequent agent trusts the previous agent's output, and by the time problems are noticed the root cause is buried multiple layers deep' — the canonical case for per-step (span-level) verification rather than only final-output checking.

**Cost / reliability pain:**
- The compounding-error math every builder feels but can't fix: at 95% per-step accuracy a 10-step workflow is only ~60% reliable end-to-end; even 99% per-step decays to 90.4% at 10 steps and 36.6% at 100 steps. Per-step accuracy and workflow success are NOT the same number — this is THE slide the room nods at.
- Silent failures: agents 'return a confident, well-formatted output while getting the answer completely wrong.' Schema validation says 'valid,' the demo looks great, and the bug ships. Teams have no cheap signal to catch the wrong-but-well-formatted case.
- Runaway token bills with no per-step attribution: $4,200 over a weekend on an autonomous refactor; heavy agent users $1,500-$3,000/mo without guardrails vs $400-$700 with; agents burn 50-100x a chat because 62% of the bill is re-sent context. Founders can't see WHICH step is burning the money.
- The false economy of cheap models: 'if a more capable model produces correct output on the first attempt where a cheaper model requires retries, the effective cost difference shrinks substantially.' Teams pick the cheap model to save money and end up paying more in retries, validation overhead, and bad-output cleanup — but they have no instrumentation proving it.
